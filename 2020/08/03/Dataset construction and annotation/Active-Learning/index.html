Active Learning

Posted on 2020-08-03 | In Dataset construction and annotation , Active Learning

Overview of Active Learning

  • Inspirated Scenario
    To classify a large amount of unlabeled data, but there is no labeled data available to train the classifier, and due to time / money cost constraints, it is difficult to label a large number of data sets for training the classifier.

  • The key hypothesis
    Whether the learning algorithm is allowed to choose the data from which it learns—to be “curious,” if you will—it will perform better with less training.

  • Goal
    Reduce labeling costs.

  • Sample Information - Entropy
    Before introducing the concept of active learning, we can first talk about sample information.
    Sample information can simply means: the information that each sample from training dataset brings to the training model is different. That is, each sample has a large or small different contribution to training model.

    And we can quantify this information by Entropy. Normally the higher entropy, the larger information will be.

  • Definition of Active Learning
    In some cases, the unlabeled data is quite abundant and the labeled data is quite rare, and the cost of manually labeling the data is quite high (both time and money). In this case, we can let the learning algorithm proactively propose which data should be labeled, and then we can send these data for labeling by experts, and finally add these new labeled data to the training dataset to train the algorithm. This process is called Active Learning.

Active learning VS semi-supervised learning

Many people think that active learning also belongs to semi-supervised learning, but in fact it is different. Semi-supervised learning, transductive learning and active learning are all learning techniques that use unlabeled data, but the principal idea behind them is different.

As mentioned above, the “active” of active learning refers to be initiative to make a labeling request, that is to say, an external entity (usually a person in the relevant field) that can label the data is required, which means, active learning is an interactive process.

Semi-supervised learning specifically refers to only use unlabeled data based on the learning algorithm self without manual intervention.

Active learning Scenarios

  • Membership query synthesis, i.e., a generated sample is sent to an oracle for labeling.
    This is a big term which simply means that the learner generates/constructs an instance (from some underlying natural distribution). For example, if the data is pictures of digits, the learner would create an image that is similar to a digit (it might be rotated or some piece of the digit excluded) and this created image is sent to the oracle to label.

  • Stream-Based selective sampling, i.e, each sample is considered separately - in our case for lable-querying or rejection. Similarly to online-learning, the data is not saved, there are no assumptions on data distribution, and therefore it is adaptive to change.

  • Pool-Based sampling, i.e., sampled are chosen from a pool of unlabeled data for the purpose of labeling. In this tutorial we use the third scenario. This setting assumes that there is a large pool of unlabelled data, as with the stream-based selective sampling. Instances are then drawn from the pool according to some informativeness measure. This measure is applied to all instances in the pool (or some subset if the pool is very large) and then the most informative instance(s) are selected. This is the most common scenario in the active learning community.


Composition and stages for Active learning:

  • Composition
    Active learning algorithms can be modeled by following five components:

  • Stages
    First stage: the initialization stage.
    A small part of the unlabeled samples is randomly selected and labeled by the experts as a training set to build an initial classifier model;
    The second stage: the cyclic query stage.
    In the unlabeled sample set $𝑈$, certain unlabeled samples are selected for labeling and added to the training sample set $𝐿$ according to a certain query standard $𝑄$, and the classifier is retrained until the training stop criteria is reached.



Concrete steps for Pool-Based sampling scenario

  1. Divide the data to a ‘pool’ and a test-set
  2. Select ‘k’ samples from the pool for the initial train-set and label them, the remaining data will be the validation-set
  3. Normalize all the sets
  4. Train the Model using the train-set, with balanced weights.
  5. Use the trained model with the validation-set, get probabilities per sample.
  6. Use the trained model with the test-set, get performance measures.
  7. Select ‘k’ most-informative samples based on per-sample-probabilities, i.e., those that the model was most uncertain about regarding their labelling.
  8. Move these ‘k’ samples from the validation set to the train-set and query their labels.
  9. Inverse normalization for all the data-sets
  10. Stop according to the stop criterion, otherwise go to 3.

Query / Selection Strategy

I will use the table below to explain the query strategies. This table shows two data points (instances) and the probabilities that each instance has each label.

  • Random selection: we just select ‘k’ random samples from the unlabeled dataset.

  • Least Confidence (LC): in this strategy, the learner selects the instance for which it has the least confidence in its most likely label. Looking at the table, the leaner is pretty confident about the label for d1, since it thinks it should be labelled A with probability 0.9, however, it is less sure about the label of d2 since its probabilities are more spread and it thinks that it should be labelled B with a probability of only 0.5. Thus, using least confidence, the learner would select d2 to query it’s actual label.

  • Margin Sampling: the shortcoming of the LC strategy, is that it only takes into consideration the most probable label and disregards the other label probabilities. The margin sampling strategy seeks to overcome this disadvantage by selecting the instance that has the smallest difference between the first and second most probable labels. Looking at d1, the difference between its first and second most probable labels is 0.81 (0.9 - 0.09) and for d2 it is 0.2 (0.5 - 0.3). Hence, the leaner will select d2 again.

  • Entropy Sampling: in order to utilize all the possible label probabilities, we can also use a popular measure called entropy. The entropy formula is applied to each instance and the instance with the largest value is queried.

Stopping criteria

  • All unlabeled data is running out.

  • No money or access for further labeling.
